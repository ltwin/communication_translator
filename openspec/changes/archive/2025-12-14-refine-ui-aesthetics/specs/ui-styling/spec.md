# ui-styling Specification

## 新增需求

### 需求：简洁色彩系统

系统**必须**采用黑白灰为主的简洁色彩方案，通过CSS变量实现主题化。

#### 场景：亮色模式配色

**假设** 用户选择亮色模式
**当** 页面渲染完成
**则** 页面应显示白色背景（#ffffff）、深色文字（#1a1a1a）、浅灰边框（#e5e5e5）

#### 场景：暗色模式配色

**假设** 用户选择暗色模式
**当** 页面渲染完成
**则** 页面应显示深色背景（#121212）、浅色文字（#f5f5f5）、深灰边框（#2d2d2d）

#### 场景：功能色应用

**假设** 系统需要显示状态信息
**当** 显示成功状态时
**则** 应使用绿色（#16a34a），在两种模式下保持一致
**当** 显示错误状态时
**则** 应使用红色（#dc2626），在两种模式下保持一致

---

### 需求：主题切换功能

系统**必须**提供亮色/暗色模式切换功能，并记住用户偏好。

#### 场景：主题切换按钮

**假设** 用户访问翻译助手页面
**当** 页面渲染完成
**则** header区域应显示主题切换按钮

#### 场景：点击切换主题

**假设** 当前为亮色模式
**当** 用户点击主题切换按钮
**则** 页面应立即切换为暗色模式，按钮图标相应更新

#### 场景：主题持久化

**假设** 用户已切换至暗色模式
**当** 用户刷新页面或重新访问
**则** 页面应自动应用暗色模式

#### 场景：跟随系统偏好

**假设** 用户首次访问且未手动设置主题
**当** 页面加载时
**则** 系统应检测用户系统的颜色偏好（prefers-color-scheme）并应用对应主题

---

### 需求：无emoji界面

界面**必须**不使用emoji字符，采用纯文字或SVG图标替代。

#### 场景：页面标题

**假设** 用户访问翻译助手页面
**当** 页面渲染完成
**则** 页面标题应显示纯文字"沟通翻译助手"，不包含emoji

#### 场景：选项文字

**假设** 用户查看翻译方向选择器
**当** 下拉菜单展开时
**则** 所有选项应为纯文字描述，不包含emoji

#### 场景：按钮文字

**假设** 用户查看操作按钮
**当** 页面渲染完成
**则** 复制按钮应显示纯文字"复制"，复制成功后显示"已复制"

#### 场景：动态内容

**假设** 系统返回智能识别结果
**当** 显示识别元数据时
**则** 标签应为纯文字"智能识别:"，不包含emoji

---

### 需求：视觉层次清晰

界面**必须**通过颜色深浅和间距建立清晰的视觉层次，在两种主题下均保持良好可读性。

#### 场景：头部区域

**假设** 用户访问翻译助手页面
**当** 页面渲染完成
**则** header应与主体内容通过视觉元素（如分隔线或背景色差异）区分

#### 场景：按钮突出

**假设** 用户查看翻译按钮
**当** 页面渲染完成
**则** 主操作按钮应通过对比色形成视觉焦点（亮色模式用深色按钮，暗色模式用浅色按钮）

#### 场景：输入输出区分

**假设** 用户查看输入和输出区域
**当** 页面渲染完成
**则** 两个区域应通过不同的背景色或边框样式区分
