# 提案：UI审美优化

## 背景

当前UI采用紫色渐变主题色（#667eea → #764ba2），并大量使用emoji图标，整体风格偏"AI生成感"。用户希望提升UI审美，采用更简洁、高级的设计语言，并支持暗色/亮色模式切换。

## 目标

1. **去除emoji依赖**：用简洁的文字替代emoji
2. **色调优化**：从紫色渐变改为黑白灰高级色调
3. **视觉简化**：减少视觉噪音，突出内容本身
4. **主题切换**：支持亮色模式和暗色模式

## 变更范围

| 文件 | 变更内容 |
|------|----------|
| `static/index.html` | 移除emoji，添加主题切换按钮 |
| `static/style.css` | CSS变量系统，亮/暗双主题样式 |
| `static/app.js` | 移除emoji，添加主题切换逻辑 |

## 设计方案

### 色彩系统（CSS变量）

#### 亮色模式 (Light)

| 变量 | 值 | 用途 |
|------|------|------|
| `--bg-primary` | `#ffffff` | 页面背景 |
| `--bg-secondary` | `#f5f5f5` | 卡片/输入框背景 |
| `--bg-tertiary` | `#e5e5e5` | 边框/分隔线 |
| `--text-primary` | `#1a1a1a` | 主要文字 |
| `--text-secondary` | `#666666` | 次要文字 |
| `--text-muted` | `#999999` | 弱化文字 |
| `--accent` | `#1a1a1a` | 强调色（按钮等） |
| `--accent-hover` | `#333333` | 强调色悬停 |

#### 暗色模式 (Dark)

| 变量 | 值 | 用途 |
|------|------|------|
| `--bg-primary` | `#121212` | 页面背景 |
| `--bg-secondary` | `#1e1e1e` | 卡片/输入框背景 |
| `--bg-tertiary` | `#2d2d2d` | 边框/分隔线 |
| `--text-primary` | `#f5f5f5` | 主要文字 |
| `--text-secondary` | `#a0a0a0` | 次要文字 |
| `--text-muted` | `#666666` | 弱化文字 |
| `--accent` | `#ffffff` | 强调色（按钮等） |
| `--accent-hover` | `#e0e0e0` | 强调色悬停 |

#### 通用色（两种模式共享）

| 变量 | 值 | 用途 |
|------|------|------|
| `--success` | `#16a34a` | 成功状态 |
| `--error` | `#dc2626` | 错误状态 |
| `--warning` | `#d97706` | 警告状态 |

### 主题切换方案

1. **切换按钮位置**：header右侧，使用简洁的图标（☀/☾ 或文字"亮/暗"）
2. **实现方式**：通过 `<html data-theme="light|dark">` 属性切换
3. **持久化**：使用 localStorage 记住用户选择
4. **默认行为**：跟随系统偏好（`prefers-color-scheme`）

## 非目标

- 不改变布局结构
- 不改变功能逻辑
- 不改变响应式断点

## 验收标准

1. 页面无任何emoji字符
2. 亮色模式：黑白灰色调，白色背景
3. 暗色模式：深色背景，浅色文字
4. 主题切换按钮可正常切换模式
5. 刷新页面后主题选择被记住
6. 首次访问跟随系统偏好
7. 所有交互状态（hover、focus、disabled）在两种模式下正常
8. 移动端适配不受影响
